Adding Backgrounds

The secret of good web design is drawing the visitors eyes to important parts of the page. Background colors and images help to break up the page and focus attention. Although some images are an important part of the page content, and need to be embedded in the HTML using <img> tags, others are simply decorative touches that should be addes through CSS. Using CSS for background images not only keeps your HTML markup less cluttered and easier to maintain, but it also gives you considerable amount of control over position and size.

Controlling Backgrounds with CSS

CSS Background Properties
Property Initial Value Description
background-color transparent Sets the background color of an element
background-image none Sets the background image of an element
background-repeat repeat Controls how background images repeat if smaller than the element. By default, images tile horizontally and vertically
background-attachment scroll Determins wheter the background image remains in a fixed position or scrolls with the page
background-position 0% 0% Determines the background images horizontal and vertical position. The default is the top left of the element
background-origin padding-box Determines the point from which background position is measured
background-clip border-box Determines the area covered by the background
background-size auto Controls the size of background images
background See individual properties Shorthand property for setting all background properties in a single declaration

Apart from background-color, all the individual properties are concerned with background images. The background properties are not inherited, but the background of a parent element remains visible because the default of backround-color is transparent.

Changing the Background Color of an Element

As you might expect, the background color property takes as its value a color, which you can be expressed in any of the ways described in Setting Color Values in Size, Colors and Files namely:

  • A 3 or 6 digit hexadecimal number
  • A color keyword
  • An rgb() value
  • An hsl() value
  • A color with alpha transparency specified as an rgba() or hsla() value

Setting a Background Color with Alpha Transparency

Modern browsers support alpha transparency, which creates a translucent effect that can produce a more pleasant result than a solid block of color in some circumstances. Using Background Images

The remaining background properties deal with adding and controlling the behavior of background images. Theres also the background shorthand property, which allows you to define all aspecs of an elements background (including color) in a single declaration. First, lets take a look at how you add background images.

Adding Background Images

The background-image property defines the source of an elements background images. It accepts two possible values, namely:

  • url() The path to the images goes between the parenthesis as described in future
  • none No background image. This is the default value.

All browsers in widesprean support multiple background images on the same element. To specify more than one image, seperate each url() value with comma. if the background image overlap, the first one is drawn on top with each subsequent one behing previous one.

Controlling How Background Images Repeat

The property that controls the way background images are repeated is called background-repeat.

Values That Work in All Browsers

The background-repeat property accepts the following values in all browsers:

  • repeat This is the default value. It repeats the image both horizontally and vertically to fill the available space
  • repeat-x This repeats the image horizontally only
  • repeat-y This repeats the image vertically only
  • no-repeat Display the image once only

Setting the value of background-repeat to repeat-x tiles the background image across the horizontala axis only. This can be useful for creating a gradient background. The background image needs to be only a thin slice. For example, the background image below is only 20px wide. If you set the background color to the same as bottom of the image, it gives the appearance of an continous color. The style rule for <body> below looks like this:

New Features in CSS3

In CSS2.1 background images simply tile in the specified direction and are cut off at the edge if they dont fit exactly. CSS3 addresses this issue with two new values for the background-repeat property. Both repeat the background image a sufficient number of times to cover the available space. But they differ in how they then treat the background.

  • space Any space left over is evenly distribution between the images
  • round The images are rescaled to fit the background

Controlling Background images During Scrolling

The background-attachment property controls what heppens to background images when the page or element is scrolled. It accepts the following values:

  • scroll This is the default value. The background is fixed with regard to the element, so it scrolls with the page
  • fixed The background is fixed in relation to the browser viewport, so it remains in the same position when the page is scrolled
  • local The background is fixed in relation to the elements contents. If the element has scrollbar, the background scrolls with the content.

Controlling the Position of Background Images

By default, browsers place the top-left corncer of background images a the top left of the element, and apply the background to the whole element(unless you restrict it using background-repeat). However, the background-position property allows you to specify new offsets for the background image to place it exactly where you want. CSS3 introduces two other properties: background-origin and background-clip, which give you control over where the offlsets are measured from and how far the background extends.

You specify the offsets for the background-position using any combination of the following:

  • Lenghts,such as pixels and ems
  • Keywords
  • Percentages
Keywords and Percentage Values for the background-position Property
Keyword Percentage Equivalent
left 0%
center 50%
right 100%
top 0
bottom 100%

Syntax

The background-position property expects one or two values:

  • If you use one value, the other value is automatically set to center or 50%
  • If you use two keywords, the horizontal and vertical values can be in either order
  • If you use any other combination of two values, the first one must specify the horizontal offset and the second one the vertical offset

When using two values, make a habit of always putting them in order: horizontal, vertical. Although specifying background-position as top is valid, 0% right is not.

background-position: left bottom; background-position: 30% 40%; background-position: 60px 900px;

Settint the Size of a Background Images

When adding a background image to an element, its important to realize that the background is simply that. The element wont increase its size to accommodate that great image youve chosen, If you want the image to be displayed in full, you need to make sure the element is wide and tall enough. At least, thats the case in older browsers. CSS3 gives you more flexibility with the background-size property, which supported by all browsers in widespread uses.

The background-size property accepts the folling value: contain Scale the image to the largest possible size so that both its width and height fit into the background positionig area. cover Scale the image to its smallest size so that both its width and height can comepletely cover the background positioning area. One or two lengths or percentages When using lengths or percentages, the first value sets the width of the backgroud as auto. Percentage are relative to the background positioning area as determined by background-origin. The contain, cover, and auto keywords all preserve the images aspect ratio.

The same background images is resized differently without losing its aspect ratio. Bot images containers have a dimention of 445y by 425x.

Using the background Shorthand Property

Instead of individual properties, you can wrap the definition in a single declaration using the background shorthand property. All browsers support CSS2.1 syntax, which combines the folling properties:

background-color background-image background-repeat background-attachment background-position

Values can be in any order. Omitted values use the initial value listed at the beginning tables. For example, the body style rule below contains four background properties:

body{ background-color: #FFF; background-image: url(yawa/animal/piste.jpeg); background-repeat: no-repeat; background-attachment: fixed; }

They can be combines in a single shorthand rule as a space-seperated list like this:

body{ background: url(yawa/animal/piste.jpeg) fixed no-repeat #FFF; }

In this example, theres no value for background-position, so it defaults to 0%, 0% in other words, the top left of an element.

Caustion The backgruond shorthand property automatically resets all values to their defaults before assigning the values given in the declaration. This can have unexpected conseqiences if you use the shorthand version at the end of a style rule that already contains individual background properties. Dont use background as quick way of adding a single value, such as color. The shorthand is great for grouping several values, but its easy to override existing values by mistake.

Adding Multiple Background Images to an element

CSS3 allows you to add multiple background images to an element by listing the image sources as a comma seperated list. The firs image in the list is displayed at the front, and all subseqent images are displayed behind. Obviosly, adding multiple images to an element only makes send if you control the default repeat pattern and other background properties. Consequently, in CSS3 compaint browsers all background properties except backgroud-color accept a comma-seperated list of value, which need to be in the same order as the images. If a property has only one value, the same value applies to all images.

height: 550px; width: 550px; background-color: orange; background-image: url(./imgex/RV.png),url(./imgex/RV.png); background-position: center; background-repeat: no-repeat, repeat; background-size: auto,auto; background-origin: padding-box, border-box; background-clip: content-box, border-box; border-radius: 50px;

This adds RV.jpeg as the first image, and with RV.jpeg behind. Theres only one value for backround-position, so it applies both to images. In the remaining decalrations, the first value applies to RV.jpeg, and the second one to RV.jpeg. The values for RV.jpeg rest its size, prevent it from repeating and chage its background positioning and painting areas. The other image, RV.jpeg is automatically tiled and covers the whole element.
