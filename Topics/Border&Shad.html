Adding Borders and Drop Shadows

Creating Basic Borders

Because CSS gives you control over the color, style, and width of each side of an element, there are no fewer than 20 properties just for basic borders. The reason there are so many is to reduce the amount of typing. Most properties are shorthand version. Once you know the basic principles, you can choose whichever best suits your needs. However, lets start by looking at the individual properties.

Setting Indiviual Border Properties

Indiviual properties allow you to set each aspect of an elements borders independently. Below lists the individiual border properties that are supported by all browsers in current use.

The color properties accept any color value, but for cross-browser compability, its best to use hexadecimal notation, a color keyword, or rgb(). If you specify a color with alpha transparency using rbga() or hsl() some ignore(old reference) and color and use the text color instead. To specify an opaque fallback color for older browsers, you need to use shorthand properties as descrbied later. Browsers also use the text color if you dont set a color for the border.

The style properties accept any of the following kewords: none, hidden, dotted, dashed, solid, double, groove, ridge, inset, or outset. Using Border Shorthand Properties

To avoid unneccesarrily verbose style rules, CSS provides not one, but eight shorthand properties for definig borders, as listed below.

Border Shorthand Properties
Property Description
border-color Defines the color of each border
border-style Defines the style of each border
border-width Defines the with of each border
border-top Sets the color, style, and with of the top border
border-right Sets the color, style, and width of the right border
border-left Sets the color, style, and width of the left border
border-bottom Sets the color, style, and with of the bottom border
border Sets the same color, style, and width for all four borders.

The border-color, border-style, and border-width shorthand properties accepts up to four values following the same formula as the shorthand for magins and padding:

One value: Applies equally to all four sides Two values: The first applies to the top and bottom, and the second wo the left and right Three values: The first applies to the top, the second to the left and right, and the third to the bottom Four values: The values are applied in clockwise order starting from top

The border-top, border-right, border-bottom, and border-left shorthand properties accpet a space seperated list of the color, style,and width values that you want to apply to that side. The values can be in any order.

h1{ border-left: teal solid 20px; }

The border shorthand property also takes a space seperated list of color, stlye, and width values, which can be in any order. The same value is applied to all four sides of the element. If a value isnt specified, the initial value for the individual property is applied. If you want the same border on all four sides, the border shorthand is clearly the best option. For example, the following definition creats a 4px double red border around an element:

border: 4px double #F00;

Adding Round Corners and Drop Shadows Without Images

Creating rounded corners and top shadows once took a lot of patient effor, but now takes just a few seconds.

Creating Rounded Corners With border-radius

If youre tired of being boxed in by angular designs, the border-radius and related properties might be just what youve been waiting for. In spite of the name, an element doestn need to have a vorder for your to give it rounded corners. You can adjust each corner individuwally or apply the same values to each one. Whats more, you can set the horizontaland vertical radii of each corner seperatelym so yore not limited to uniform shaped. This fine degree of control can make the syntax seem overhelming at first, but its not difficult. Whats more, there are visual tools available to do the calculations for you.

Rounded Corner Properties
Property Initial Value Description
border-top-left-radius 0 Rounds an elements top-left corner
border-top-right-radius 0 Rounds the top-right corner
border-bottom-left-radius 0 Rounds the bottom-left corner
border-bottom-right-radius 0 Rounds the bottom right corner
border-radius Sets the values for all corners in a single declaration

Rounding Individual Corners

The individiual individiual properties take one or two lenghts or percentages, which are used to calclate the curve applied to the corner. Percentages are relative to both the elements width and height.

  • A single length creates an evenly rounded curve
  • A single percentage creates an evenly rounded curve only if the element is square. Otherwise, the browser calculates the percentage values of the width and height, and treats them as two values
  • With two values, the first sets the horizontal radius of a quarter ellipse and the second sets the vertical radius
  • If any value is 0, the corner is square

In the following example, each corner is set using different method.

#box{ height: 200px; width: 200px; border-top-left-radius: 25px; border-top-right-radies: 30%; border-bottom-left-radius: 50% 30%; border-bottom-right-radies: 100px 45px; }

The top-left corner is shaped using a single length, which produces an evenly rounded curve. The top-right corner uses a single percentage, but the element isnt square. So, the browser calculates 30% of the width (60px) and 30% of the height (45px). The resulting values are used as the horizontal and vertical radii of a quarter ellipse. The bottom two corners create identical curves using percentage ang lengths.

Using border-radius shorthand

The border-radius shorthand property sets all four corners in a single declaration. Using it can be either freindlishly complex or blissfully simple. Lets start with a simple example, which is probably the way most people use it.

background-color: teal; width: 200px; height: 150px; border-radius: 25px;
background-color: teal; width: 150px; height: 150px; border-radius: 50%;

Using border-radius with a single value applies the same curve to all corners

Using border-radius with more than one value is less straightforward. First lets deal with using the same value for the horizontal and vertical radii of each curve.

Two values: The first applies to the top-left and bottom-right corners. The second applies to the top-right and bottom-left corners. Three values: The first applies to the top-left corner. The second applies to the top-right and bottom-left corners. The third applies to bottom-right corner. Four values: The values are applied to each corner in a vlockwise direction starting from top left.
Its when you want to use different values for horizontal and vertical radii that the border-radius shorthand becomes mind-mumbling complex, taking 2-8 values. Unless you want to apply the same values to each other, its brobably much easier to use the individual properties for each corner. Still, Ill explain how it works and let you make up your own mind. The values for the horizontal and vertical raii are seperated by a forward slash. However, you dont specify each pair seperately. All horizontal values come before the slash, and all vertical value come after. And this is where the syntax becomes really complex, you dont need to use the same number of values before and after the slash. So, for example, you could have three horizontal values before the lash and two vertical ones after slash. The rules regarding which corners the values apply to are the same as just described for two, three and four values.

The folling styles for box1 using border-radius shorthand have exactly the same meaning as the individual rules for box2 below

.box1{ border-radius: 20% 10% 40% / 15% 30%; } .box2{ border-top-left-radius: 20% 15%; border-top-right-radius: 10% 30%; border-bottom-right-radius: 40% 15%; border-bottom-left-radius: 10% 30%; }

Adding Drop Shadows With box-shadow

The box-shadow property adds one or more drop shadow to an element. You specify a shadow using 2-4 space seperated lenghts, which represent the following values(in this order).

Horizontal offset: A positive value positions the shadow to the right. A negative value positions it to the left. Vertical offset: A positive value moves the shadow down. A negative value moves it up. Blur radius: If ommited or set to 0, the shadow has a crisp edge. Negative values are not allowed. Spread distance: A positive value causes the shape to expand in all directions. A negative value (used with an inset shadow) causes it to contract. To create a spread effect without setting a blue radius, the third length must be set to 0
In addition to these lengths, you can specify a color. Otherwise, the shadow is the same color as elements text. By default, the shadow is drawn outside the element, but you can draw it inside the elmeent by adding the inset keyword. To specify multilple shadwos on the same element, seperate the definition of each shadow with a comma. The shadow effects are drwon with the first one on top, and each subsequent one layered behin. If you want shadows to overlap, list the smaller ones first.

drop{ box-shadow: 5px 5px 7px rbga(0,0,0,0.3); }

The next two rules add the inset keyword:

Positive Inset
Negative Inset
.positiveInset{ box-shadow: inset 10px 10px 10px rgba(0,0,0,0.3); } .negativeInset{ box-shadow: inset -10px -10px 10px rbga(0,0,0,0.3); }

Simulating Multicolored Borders

Combining a spread shape with offsets and a blur radius rarely produces an attractive effect. However, using multiple spread shapes with no offset or blur offers an easy way to create multicolored borders. The box-shadow property accepts multiple shadow effects as a comma sperated list. Each subsequent effect is painted behind the precedeing one, so the smallest one mus come first followed by the others in increasing size. The style rule for this looks like this:

Multiple Borders
text-align: center; width: 200px; height: 20px; background-color: magenta; border-radius: 20px; box-shadow: #a0d5d6 0 0 0 5px, #789aa1 0 0 0 10px, #ad9a27 0 0 0 15px, #304345 0 0 0 20px

The horizontal and vertical offsets are blur radius in each value are set to 0. The spread value in each one is increamented by 5px, producing the result seen above. the div has rounded corners, to the spread colors added by box-shadow follow the same outline.

Using Border Images

For a more decorative border, you need to use an image. In CSS2.1, this involves creating up to nine images and wrapping the element in a complex series of div elements. In CSS3 simplifies the process by using just in image, which the browser automatically slices into nine sections to build the border. The browser positions the four corner slices, and scale or tiles each side to fill the with and heigth, The dimensions of the border image arent important. You just need a design that can be sliced this way.

Individual Properties for Border Images

There are five individual properties for border images, as described below:

Individual Properties for Border Images
Property Initial Value Description
border-image-source none Specifies the image to use for borders
border-image-slice 100% Sets the inward offsets form each side of the image, dividing it into four corners, four edges, and a center section.
border-image-width 1 Sets the amount of spaces allocated to the border image on each side.
border-image-outset 0 Specifies how far the border image should extend beyond the border box.
border-image-repeat stretch Specifies how the slices are scaled and tiled along the horizontal and vertical edges.
The first two properties, border-image-source and border-image-slice, are required. The other properties are optional.

Specifying the Image and Its Slices

The border-image-source property tells the browser which image to use for the borders. You specify the image in normal way with url(). The border image slive property defines how the image should be sliced up. It expects 1-4 numbers or percentages that indicate how much of each side should be used for the border. The values are treated in a similar way to margin and padding shorthand namely:

One value: Applies to all sides Two values: The first value applies to the top and bottom, and the second to the right and left sides Three values: The first value applies to the top, the second to the right and left sides, and the third one to the bottom Four values: The values apply to each other starting from the top and moving clockwise
The unusual aspects of using numbers with border-image-slice is that they are not followed by a unit of measurement, such as px or erm. Thi is because the property is intended to be used with scalable cevtor graphics (SVG) as well as JPEG, PNG, or GIF images. When using an ordinary image, the value is the number if pixel measured from the edge that you want to use as a slice. If your using an SVG imge, the numbers are vector coordinates. Percentage values for top and bottom are relative to the images height. Those for the right and left are relative to the images width. According to the specification, the center slice should be discarded. To preserve the center slice, add fill keyword.

Controlling the Size and Position of the Border Image

The border-image-property sets the width of the slices, not the with of the border. Its a subtle, but important differences. Border images are normally displayed within the elements border box. IF you dont create a border around the element using the properties descrbed at the beginnning, there no space between the border and the padding boxes. Consequently, image wont be displayed or will be squeezed to almost nothing

The border-image-width property accepts up to four values, which are allocated to each sode in the same way as slices. The values can be specified as lengths, percentage, or numbers. A number represents a multiple of the computed border-width. Percentages are relative to the width or height of the border box for horizontal and vertical slices, respectively

Setting How the Slices Fill the Sides

By default, the browser stretches each slices so that it fills the gap between the corners. It does so without changing the height for the top and bottom slices, or the width for the right and left slices. However, the border-image-repeat property lets you specify how to handle the slices with the following keywords.

stretch This is the default. The slices are scaled along the axis to fill the gap repeat The image is tiled from the center of the side to fill the gap round The image is rescaled if neccessary to fill the gap an exact number of times space The image is repeated an exact nymber of times to fill the gap. If theres insufficient space for an exact number, the extra space is distributed evenly leaving a gap between individual tiles.
If you use on keyword, it applies to all sides. If you use two keywords, the first one applies to the top and bottom, and the second applies to the right and left.
border-image-repeat: stretch round;

Using the Shorthand Property for Border Images

The shorthand border-image property combines the five individual properties in a single declaration.

Full Shorthand Syntax

border-image: source slices / width / outset repeat;

As explained before, only the image source and slices are required. The other three values are optional. However, note the following rules:

  • If you include 1-4 values for width, they must follow the values for slices, seperated by a forward slash
  • To include 1-4 values for outset, they must follow the width values, seperated by a forward slash
  • You cannot specify values for outset without also setting width
  • If you omit width and outset, there should be no slash between any of the values

width: 300px; height: 300px; color: magenta; border-image: url(./imgex/RvFrame.png); border-image-slice: 24 fill; border-image-width: 18px 12px 24px; border-image-outset: 6px 12px 12px; border-image-repeat: round stretch;
width: 300px; height: 300px; color: magenta; border-image: url(./imgex/RvFrame.png) 24 fill / 18px 12px 24px / 6px 12px 12px round stretch;
