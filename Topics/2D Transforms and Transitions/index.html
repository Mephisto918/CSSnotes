Changing an Elements Position with translate()

The translate() function and its sinlge-axis counterpart translateX() and translatey() have a similar effect as relative positioning. They move an element from its default position without affecting the position of surrounding elements. Used in combination with other transfrom function, they can move an element at the same time as scaling, rotating, or skewing it.

Rotating Elements

To rotate an element, use the transform property and set its value with rotate() function, which takes an angle as its argument. You can express the angle using degrees, radians, gradians, or turns. Degrees and turns are the easiest to use. One turn is equal to 350deg.

Scaling Elements

To scale an element, use the transform property and set its value with the scaleX() scaleY() scale() functions. The first two functions take a number as their sole argument, and use it as a scaling factor for the X or Y axis, respectively. For example, below contains three identical images. By default, scaling is done from the center of an element.

The scale() function accepts one or two arguments. If you pass it a single number, the same scaling factor is applied to both axes. If you pass it two numbers seperated by a comma, the first is used as the scaling factor for X axis and the second for the Y axis

Flipping Elements

The table above does not contain a transform function to flip elements horizontally or vertically, but you can achieve the same effect by passing negative values to scaleX() scaleY() scale(). Setting the value to -1 preserves the original size, but flips the element on the relevant axis.

See inspect tool

Skewing Elements

To skew an element, use the transform property and set its values by passing an angle as the argument to skewX() skewY. With skewX(), a positive angle skews the element to the left, and negative angle skews it to the right. With skewY(), a positive angle skews the elements vertically and rotates it in a clockwise direction. A negative angle rotates it in an anticlockwise direction.

Use inspect tool

Transforming Elements with the matrix() Function

The matrix() function specifies a 2D transform in a form of a transformation matrix (http://en.wikipedia.org/wiki/Transformation_matrix). The function requires siz arguments, all of which are numbers seperated by commas.

Tip Unless you are an expert in linear algebra (ako kay dele haha), you can forget about the matrix() function. Although it allows you to combine multiple transforms in a single operation, everthing that the matrix() function does can be achieved using the individual transform function described in the precedeing topics

The last two arguments always specify the number of pixels you want to move (translate) the element along the horizontaland vertical axes, respectively. The first four arguments control other transforms.

In most cases, the first and fourth arguments are the scaling factors for the horizontal and vertical axes, respectively. The second argument usually controls the angle of vertical skew, while the third one controls horizontal skew. All four arguments are used in combination to rotate the element. What makes using the matrix() function complicated is the need to convert angles of skew and rotation using trigonometric functions.

pecifying the Duration of the Transition

You specify the duration of the transition with transition-duration property, which expects a single duration or comma-seperated list of durations. Times can be set in seconds or thousandths of a second (ms).

If you specify just one value, the same duration apllies to all properties being transitioned. If you supply multiple durations, they are applied to the properties in the same order as they are listed in transition-property.

Setting the Pace of the Transition

The transition-timing-function property controls how intermediate values are calculated during the transition. The property accepts one or more of the following values as a comma-seperated list:

  • ease Starts slow, speeds up, and slows towards the end
  • linear Transitions at an even pace throughout
  • ease-inStarts slow and speeds up
  • ease-out Starts fast and slows towards the end
  • ease-in-out Similar to ease, but slower at each end
  • cubic-bezier() Specifies a custom transition using a cubic-bezier curve
  • step-start The change takes place at the beginning of the duration
  • step-end The change takes place at the end of the duration
  • steps() The change takes place in the specified number of equal stpes

As with transition-duration, a single value is applied to all transition. If you specify multiple values as a comma-seperated list, they are applied in the same order to the individual propertues listed in transition-proerty. Excess values are ignored, and if you specify fewer transition-timing-function values than properties, the browser cycle through them.

The first five keywords in the preceeeding list are consntants for easing functions and based on a cubic-bezier curve. If you have used Flash, you will be famillar with the concept of easing, which varies the pace of a transition(or tweenm as its called Flash). The cubic-bezier() function allows you to customize the pace. It takes four arguments like border-top-right-radius

cubic-bezier(x1, y1, x2, y2);
Both x1 and x2 must be in the range 0-1. The y1 and y2 values can exceed that range.



The steps() function takes two arguments: the number of discrete steps in the transition and the keyword start or end. If you omit the second argument, it defualts to end. Each changes color at a different pace

div { width: 100px; height: 100px; background-color: black; border: 1px solid white; transition-duration: 2s; } #box1{ transition-property: background-color; transition-timing-function: steps(5, start); } #box2{ transition-property: background-color; transition-timing-function: steps(5, end); } #box3{ transition-property: background-color; transition-timing-function: step-start; } #box4{ transition-property: background-color; transition-timing-function: step-end; }

The transition-duration property is set to a deliberately long five seconds. The first two elements use the steps() function to change the color in five discrete steps. However, the timing is affected by the second argument. The first div uses start, to the color immediately changes to the first intermediate shade. In the next div, the second argument is end. As a result, the color does not change until the first second has passed.
The last two div elements use the step-start and step-end constants. The first changes color immediately on mouse over and mouseout. The last one does not change unless the mouse pointer is held over for full 2 seconds. When you mouse away from div. it does not rever to white until two seconds have passed.

Delaying the Transition

By default, transitions starts immediately, but you can use the transition-delay property to delay the whole or part of a transition. Like the other transition properties, it accepts a single time value or a comma-seperated list of values. A single value applies to all properties being transitioned. A list of values is applied to properties in the same order as the are listed in transition-property

Delaying a propertys transition by the same amount as the previous propertys duration creates a sequence of changes. For example, the styles change the background color, width, and height of a box on hover like this:

#delay { width: 100px; height: 100px; background-color: blue; margin: 10px; transition-property: background-color, width, height; transition-duration: 1s; transition-delay: 0s, 1s, 2s; } #delay:hover{ background-color: yellow; width: 150px; height: 150px; }

Changing the Reverse Transition

As the example from before, transition delays are executed in the same order as when the element returns to its normal state. In many cases, this is unlikely to cause a problem. But if you want to run a sequence of transition-reverse, its quite easy to do.

All thats needed is to define transition-delay in two places

  • The timings for the forward sequence go in the style rule that define the state after the transition, for example in the :hover pseudo-class
  • The timings for the reverse sequence go in the style rule that defines the elements normal state

#delay-rev { width: 100px; height: 100px; background-color: blue; margin: 10px; transition-property: background-color, width, height; transition-duration: 1s; transition-delay: 0s, 1s, 2s; } #delay-rev:hover{ background-color: yellow; width: 150px; height: 150px; transition-delay: 2s, 1s, 0s; }

in #delay-rev:hover I added transition-delay: 2s, 1s, 0s; which reversed in the original transition-delay values

Using the Shorthand Property

Instead of using the individual properties, you can define a transition in a single declaration with the transition shorthand property. For a single property, just list the values seperated by spaces, For example, the following applies a one-second linear transition to background-color, delayed by a quarted of a second:

transition: background-color 1s 250ms linear;

To jump to the end state immediately after a delay, use a single time value equal to the length of the delay, and set the timing function to step-end is 0s

To apply transition values to all properties, just set the values you want to change from the initial values listed before. For example, if you want all properties to transition over half a second without delay, this is all you need:

transition: 500ms;

Alternatively, if you want to make it clear that all properties are affected, inlcude all keyword:

transition: all 500ms;

In both case, the default ease timing function will be used

To apply different values to each property, use multiple shorthand definition seperated by commas. For example the previous code to which there was delay, it can be wriiten like this

#delay-rev { width: 100px; height: 100px; background-color: blue; margin: 10px; transition-property: background-color 1s 2s, width 1s 1s, height 1s; }
