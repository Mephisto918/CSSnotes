Using font-family to Choose a Range of Alternative Fonts

The font-family property takes as its value a comma-seperated list of fonts like this:

font-family: Arial, Helvatica, sans-serif;

If any of the fonts names contain spaces, they must be wrapped in qoute like this:

font-family: "Trebuchet MS", Arial, Helvatica, sans-serif;

The browser attempts to display the first available font in the list. In the case of the previous example, the browser tries Arial if Trebuchet MS isnt available on the visitors computer. If Arials not available, it tries Helvetica. And if none of the fonts is available, it falls back to the computers default sans-serif font.

Using Lenth to set Font Sizes

Apart from minor variation between browsers, CSS pixels are fixed size. As a result, using px is much simpler than em, which is a relative measurement. The default size browsers for the text in paragraphs is 16px.

To make the text slightly smaller, you can set the font size in pixels like this:

p{
font-size: 14px;
}

To do the same with em, you need to calculate the ratio of 14px to the default 16px, in other words, 0.875. So the equivalent style rule becomes like this:

p{
font-size: 0.875em;
}

Using Percentage to Set Font size

Using a percentage value changes the font in realtion to the size inherited from any parents or ancestor elements, resulting in the same multiplier affect as with em units.

Using font-style to Italized Text

The font-style property controls whether text is rendered in italic. It accepts the following vales:

  • italic Italicize the text
  • normal Removes Italic
  • oblique Use an oblique version of the font, if one exist. Otherwise, use italic.

Because very few web-safe fonts have no oblique version, italic and normal are the only one values you need. The main use for normal is to remove italic from text that you want so emphasize inside the larger block of that text that already Italicized

Using font-weight to Make Text Bolder or Lighter

The font-wieght property accepts the following values:

  • bold Make the text bold
  • bolder Make the text bolder in relation to its parent
  • lighter Make the text lighter in the realtion to its parent
  • normal Render the text normally

One of the nine values increasing in steps at 100 from 100 to 900 (400 equals normal,and 700 uquals bold).

In practice, the only values you are likely to use are bold and normal. Setting font-weight to normal is useful when you want to display as normal text an element, such as a <th> tag (table heading), that browsers normally render in a bold font.

Using font-variant to Display Text in Small Caps

The font-variant propert accepts the following rules:

  • normal Render text using the same letter case as in the underlying code.
  • small-caps Render the text in small caps if not supported by the font. Otherwise, transform the text to uppercase.

Small caps display lowercase letters as uppercase, but in smaller size and with slightly different proportions.

Adjusting the Vertical Space Between Lines of text

The line-height property controls the vertical space or leading between lines of text. Setting the line-height propert affects the read ability of a web page. Too little and the text looks cramped and is hard to read. Too much spcae and readers wanders, making reading equally difficult.

This property accepts a lenght, percentage, or number. You can also use the keyword normal to reset line-height to its default value. Negative values are not permitted. Because of the way line-height is calculated and applied by browsers, the most consistent results are achieved by using a number without a unit of measurement like this:


line-height: 1.4;

The default value applied by the browsers varies but normally in the range of 1.0-1.2. You can use position values less than 1, but only thing smaller than 0.6 results in lines overlapping each other.

Using Shorthand font Propoerty

CSS shorthand properties lets you combine several declarations into one. Unfortunately, the font shorthand property can be difficult to remember and if you get it wrong, your srtles arent applied.

The font shorthand propert combines the following properties:

  • font-style (optional)
  • font-variant (optional)
  • font-weight(optional)
  • font-size
  • line-height(optional)
  • font-family

The syntax is difficult to remember because the first three items are optional and can be any order. However, the remaining items must come in the prescribed order. If you declare line-height it must be seperated from font size by forward slash. Take for example, the following style rule:

p{ font-style: italic; font-weight: bold; font-size: 14px; line-height 1.4; font-family: "Trebuchet MS", Arial, Helvatica, sans-serif; }br

You can convert that to shorthand like this:

p{ font: bold italic 14px/1.4 "Trebuchet MS", Arial, Helvatica, sans-serif; }

Notice that reversed the order of font-weight, which is perfectly valid. The shorthand is shorter and relatively easy to read. However, one of the most common mistakes is to leve out everything except the font-family.

Changing the Color of Text

The color of text is controlled by color property. The value is inherited so the color affects the text in all descendants elements unless a more specific rule overrides it. Its considered good practice to set the colors for background and text ath the same time beecause users can set their own default colours in a local style sheet. If your text color doesnt provide sufficient contrast with the users background, the text will illegible.

body{ font-family: Arial, Helvatica, sans-serif; color: #000; background-color: #FFF }

This sets the font color and text for the whole page, but you can override any of these properties in styles for heading, sidebars or any other part of the document.

Aligning Text Vertically

Vertical allignment is controlled by vertical-align property, perhaps one of the least well understood CSS properties. Many newcomers to CSS mistakenly believes it can alter the vertical of block elements, such as paragraphs. It cant. The vertical-align property applies to inline elements, such as images and <span> elements, and moves them in realtion to the surrounding text. The HTML <sup> <super> elements automatically shift text vertically in relation to the rest of the line. As a resilt, vertical-alignis not as useful for text as you might expect.

The property accepts the following values:

baseline Align the text to the same baseline as text in the parent element. This is the default values.

middle Align the vertical mid point 0.25em above the baseline of the parent.

sub Align text to the property position for a subscript. This does not alter the size of the font.

super Align text to the proper position for superscript. This does not alter the size of the font.

text-topAlign the top of the imaginary text box with the top of the text in the parent element.

text-bottom Align the bottom of the imaginary text box with the bottom of the text in the parent element.

top Align the top of the imaginary text box in the line with the top of parent text box

bottom Align the bottom of the imaginary text box in line with the bottom of the parents element.

inherit Vertical alignment is not inherted bt default, so use this to apply the same value as the parent element.

You can also use a length or percentage. Both align the bottom of the imaginary text box by the given amount above the bottom of the parents text box, or below it if a negative value is used. A percentage value is calculated as a proportion of the current line height.

Switching Between Uppercase and Lowercase

The text-transform property converts text to use intial caps, all lowercase, or all uppercase. Its supperted by all browsers:

capitalize - Converts to uppercase the first letter after every space

lowercase - Converts all characters to lowercase

uppercase - Converts all characters to uppercase

none - No conversion

The text-transform property is inherited, so it affects all child elements. Set the value to none in a child element to turn off an inherited text transformation rule.

Dealing with Text Thats to Long

If text is too long to fit into the space available, browsers let it overflow, destroying the design and sometimes making the text illegible. CSS lets you deal with the overflow by breaking the text or hiding it.

Breaking Overflow Text

When there are no spaces in text, such as long URL, the browser is unable to break it, resulting in an ugly overflow. All browser in current user suppoert the word-wrap property to handle this situation. It accepts the following rules:

normal - This is the default. Lines may break only at normal work break points. break-word - Lines can break at arbitrary point if no blank points can be found.

Adding Shadows to Text

The text-shadow property does exactly what you expect, it adds a shadows to text without the need to embed the text in an image. In fact, it can add multiple shadows to text. The text-shadow property expects the following value:

A length specifying the horizontal offset of the shadow. A positive value moves the shadow to the right of the text, a negative one moves it to the left. A length specifying the vertical offset of the shadow. A positive value moves the shadow below the text, a negative one moves it above. An optional length specifying the shadowns bluer radius, the shadow is the same color as the text.

The order of the three length values is fixed, but the color value can come before or after them. Although the color is optional. ommiting it usually produces unsatisfactory result. To create a realistic shadow, its best to use a color with alpha transparency. All browsers that support text-shadow also supports rgba() and hsla() color formats. The style in text-shadow contain the following class:

.no-blur{ text-shadow: 0.1em 0.1em rbga(0,0,0); } .blur{ text-shadow: 0.1em 0.1em 0.2em rgba(0,0,0,0.4); } .color-blur{ text-shadow: 0.1em 0.1em 0.2em rgba(0,102,102,0.4); }

Using Multiple Shadows for a More Intense Effect

The last two examples in text-shadows demonstrate a suprising feature of the text-shadow property. If shadow definition uses a blur radius, repeating the same definition multiple times intensifies the effect. The last two examples us the following styles:

.glow{ text-shadow: 0 0 0 0.4em #6f9; } .intense-glow{ text-shadow: 0 0 0 0.4em #6f9, 0 0 0.4em #6f9, 0 0 0.4em #6f9; }

The glow class creates a green text shadow with no offset and a 0.4em blur radius. The intense-glow class simply repeats the same definition three times, resulting in a more intense effect.
