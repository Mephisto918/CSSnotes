Universal Selector

The Universal Selector is represented by an asterisk (*). It selects any elements, but is rarely used except to create hacks to hide styles from older browsers.

Type Sekectors

A Type Selector uses the name on HTML tag without the angle brackets, and redefines the defailt look of the tag. It is also called Tag Selectors

Class Selectors

A Class Selector applies style rules to elements that have the equivalent class atribute in thier opening HTML tag. The selector is created by prefixing the class name bt a period. For example class="warning" in the opening tag.

    Class names mut also adhere to the following rules:

  • No spaces are permitted
  • The only punctuation characters permitted are the hypen (-) and underscore (_)
  • The name cannot begin with a number or a hyphen followed by a number.

ID Selectors

An ID Selector applies stles to elements that have the equivalent id attribute in thier opening HTML tag. Unlike classes, which can be applied to multiple elements in the same page. ID on different pages, as long as its only used once on each page.

The selector is creating by prefixing the ID with the hash or pound sign (#). For example, #sidebar applied to the element that has id="sidebar" in its opening tag. The rules for naming ID's are the same as for class names.

Psuedo-classes

A Psuedo-classes doesn't rely on a class attribute in the HTML markup. but is applied automatically by the browser deoending on the position at the element or its interactive state. The Psuedo-classes apply mainly to links and form elements

Psuedo-elements

Like Psuedo-classes, a Psuedo-element is not identified as such by HTML markup. It applies styles to content based on its position in the HTML herirarchy. The most widely supported Psuedo-element are:

  • [:first-letter] This applies a style to the first letter at the first line inside a block-level element, such as a paragraph as long as nothing else preccedes it.
  • [:first-line] This applies a style for the first line on the text in a block-level element. The length expands and contracts dynamincally if the text is rewrapped.

Descendant Selectors

A Descendant Selector matches any element that is a descendant of another. To create a descendant selector, separate the two selectors by a space, putting the ancestor first. For example, #sidebar p affects only paragraphs that are descendants of an element that has the ID sidebar.

Attribute Selector

An Attribute Selector matches any element that has particular attribute in its opening HTML tag. You create an attribute selector by putting the attribute-optionally with a value-in a pair of square brackets after a type (tag) selector.

The following selections described the most common attribute selectors.

  • Match Any Value

    • The simplest type of attribute selector matches an elements that contains a particular attribute in its opening tag. The values of the attribute is unimportant. For example, the following style rule adds a five-pixel red border around every image that contains the alt attribute:

    img[alt]{
    border: 5px solid red;
    }

    This can be useful for checking that all image have alternate text before uploading your site. You can see instantly if the alt attribute is missing from any image because they won't have a border on your images, that is.

  • Match an Exact Value

    • To select elements with attribute that have a specific value, add on equal sign after the attribute followed by the value you want to match. This is particularly useful when styling forms. For example, the styling rules gives all text inputs fields a width of 250 pixels.

    input type="text"{
    color: red;
    }

  • Match the Beginning of a Value

    • You can select elements bt matchmaking only the first few characters of an attribute's value. Precede the equal sign with a caret or circumflex like this:

    a [href^="http://"]{
    color: red;
    }

    This matches all links where the href attribute begins with http://. In other words, it matches all external links, except those made over a secure connection.

  • Match the End of a Value

    • To match the last few characters of an attributes value, precede the equal sign with a dollar sign. For example, the following style matches all links to PDF documents, and displays a PDF symbol alongside the link text.

    a [href$="pdf"]{
    padding-right: 20px;
    background-image: url(...../images/pdficon_small.gid);
    background-repeat: no-repeat;
    background-position: right-center;
    }

Group Multiple Selectors

To avoid repetition when applying the same styles to several elements, you can group selectors as a comma-seperated list. For example, the example style rule changes the color of all headings:

h1, h2, h3, h4, h5, h6 {
color; red;
}

What Heppens When Style Rules Conflic?

The basic principle of the cascade is that a style definition that comes after another overrides the earlier one in case of a conflict. However, not all selectors are equal. Calculating which style rule wins also depends on a concept known as Specificity.

  • Count the number of ID selectors in the selector as a
  • Count the number of class selectors psuedo-class, and attribute selectors in the selector as b
  • Count the number of type selector and psuedo-elements in the selector as c
  • Ignore the universal selector
  • String abc together to get selector's Specificity
  • Inline styles always take precedence

In other words, ID selectors have a value of 100; class, pseudo-class, and attribute selectors each have a value of 10; and type selectors and pseudo-elements have a value of 1. The CSS2.1 formula expresses each group as a comma-separated list of four numbers. The values are assigned like this:

  • Inline style: 1, 0, 0, 0
  • ID style: 0, 1, 0, 0
  • Class, Psuedo-classe, or attribute selector: 0, 0, 1, 0
  • Type selector or psuedo-element: 0, 0, 0, 1
